# 1.2 Card Parser [DONE]

## Description
Parse a card file into a structured Card object. Extracts dot-path, title, and
status from the heading; card links from frontmatter; file manifest from markdown
body. Determines node vs leaf by presence of children in frontmatter.

## File Manifest
- src/card.ts
- src/types.ts

## Acceptance Criteria
- Parses dot-path headings (e.g. "# 2.1 Plan Parser [PLAN]")
- Parses root-format headings without dot-path
- Parses all standard phases and special statuses (BLOCKED-BY, CONFLICTS-WITH, INACTIONABLE)
- Extracts file manifest list items from ## File Manifest section
- Strips frontmatter before heading search (avoids YAML comment false matches)
- Correctly identifies nodes (has children) vs leaves (no children)
- Accepts injected FileSystem for testability
