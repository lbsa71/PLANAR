# 2.3 Termination & Rate Limiting [DONE]

## Description
Implements the four termination conditions from the spec plus rate limit
handling. Parses Claude Code JSON output for cost tracking and error detection.

See @README.md — "Termination" section.

## File Manifest
- src/wrapper.ts

## Acceptance Criteria
- All cards DONE: scans plan dir each iteration
- Convergence: diffs card content before/after invocation
- Max iterations: configurable cap, default 50
- Budget exhausted: tracks cost_usd from JSON output
- Rate limit (429): detects rate_limit_error and overloaded_error, waits retry_after seconds, doesn't count as iteration
- Parses non-JSON output gracefully (falls back to plain text)
