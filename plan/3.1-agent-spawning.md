# 3.1 Agent Spawning & Reaping [DONE]

## Description
Spawn Claude Code processes for eligible cards, track completion promises at
spawn time (no listener leaks), reap finished agents, handle JSON output and
429 rate limits.

See @README.md — "Orchestrator" section.

## File Manifest
- src/orchestrator.ts
- src/types.ts

## Acceptance Criteria
- Spawns via ProcessSpawner interface (injected)
- Completion promises registered once at spawn time
- Reap removes agent from all tracking maps
- JSON output parsed for cost and rate limit detection
- Rate-limited agents get iteration count decremented (retry on next cycle)
- Handles empty and non-JSON output gracefully
