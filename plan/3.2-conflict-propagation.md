# 3.2 Conflict Propagation [DONE]

## Description
When a card has [CONFLICTS-WITH X], the orchestrator mirrors the status to card X
and regresses their common parent to [PLAN] with a conflict note. Both cards are
frozen until the parent re-decomposes.

See @README.md — "Conflict Detection" section.

## File Manifest
- src/orchestrator.ts

## Acceptance Criteria
- Mirrors conflict: if 2.1 has CONFLICTS-WITH 2.3, sets 2.3 to CONFLICTS-WITH 2.1
- Does not propagate if target already has a conflict status
- Finds common parent by dot-path prefix matching
- Regresses common parent from DONE to PLAN with conflict note content change
- Does nothing if common parent is not DONE
- Does nothing for cards with no common ancestor
