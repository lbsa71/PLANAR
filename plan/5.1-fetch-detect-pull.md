# 5.1 Fetch, Detect & Pull [DONE]

## Description
The git operations layer: fetch origin, compare HEAD to tracking branch,
capture the diff, and pull with --ff-only. If fast-forward fails, log a
warning and skip the cycle.

See @README.md — "Git Watch Mode: How It Works" steps 1-4.

This module exposes a pure function (no git side-effects by default) that
accepts injected git command runners, making it fully testable without a
real repo. The return type encodes three outcomes: `up-to-date`,
`pulled` (with diff + changed files), or `diverged` (with warning message).

Architecture: @docs/5-git-watch/ARCHITECTURE.md

## File Manifest
- src/git-ops.ts          — fetch, detect, diff, pull logic
- src/git-ops.test.ts     — unit tests with injected git runners
- src/types.ts            — GitRunner and FetchResult type definitions
- src/cli.ts              — watch command wiring (already has stub)

## Acceptance Criteria
- `git fetch origin` runs each cycle
- Detects behind state by comparing HEAD to origin/<branch>
- Captures diff and changed file list before pulling
- `git pull --ff-only` integrates changes
- Logs warning and skips on non-fast-forward (diverged history)
- Does nothing when already up to date
- All git subprocess calls are injected (dependency injection) for testability
- Return type is a discriminated union: `UpToDate | Pulled | Diverged`
