# 5.2 Impact Analysis & Card Creation [DONE]
## Description
Cross-reference changed files against every card's file manifest to find
affected cards. Create an impact card (plan/0.N-upstream-sync.md) documenting
the commit range, changed files, and affected cards with a diffstat summary.

Two functions, consumed by the watch loop after 5.1 completes a pull:

1. **`findAffectedCards(changedFiles, planDir)`** — uses `discoverCards()` and
   `parseFileManifest()` from `card.ts` to match changed files against every
   card's file manifest. Returns the list of affected cards.

2. **`createImpactCard(commitRange, changedFiles, affectedCards, diffstat, planDir)`**
   — writes `plan/0.N-upstream-sync.md` with proper YAML frontmatter and
   markdown body. Determines N by scanning existing `plan/0.*` files.

Both are pure/mechanical — no LLM reasoning involved.

See @README.md — "Git Watch Mode: How It Works" steps 5-6.

## Acceptance Criteria
- `findAffectedCards` matches changed files against all card file manifests
- Returns cards whose manifest contains at least one changed file
- Path matching is normalized (forward slashes, relative paths)
- `createImpactCard` writes a card with valid YAML frontmatter (parent: plan/root.md, root: plan/root.md)
- Impact card heading follows `# 0.N Upstream Sync [PLAN]` format
- Impact card body includes: commit range, changed file list, affected card list, diffstat summary
- Sequential impact cards get incrementing dot-paths (0.1, 0.2, ...) by scanning existing plan/0.* files
- All I/O goes through injected FileSystem for testability
- Full test coverage with dependency injection
