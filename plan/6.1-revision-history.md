# 6.1 Revision History & Frontmatter Stamping [DONE]

## Description
Add `updateFrontmatterKey` and `appendRevisionEntry` to `src/card.ts`, and expose
`lastIntegrityCheck` on the parsed `Card` type. These are the primitives used by
the integrity checker and orchestrator to maintain audit trails in card files.

## File Manifest
- src/card.ts
- src/types.ts

## Acceptance Criteria
- `updateFrontmatterKey(content, key, value)` upserts a scalar key in YAML frontmatter; creates frontmatter block if absent
- `appendRevisionEntry(content, entry)` prepends a dated bullet to `## Revision History` (newest-first); creates section if absent
- `Card.lastIntegrityCheck` is populated from `last-integrity-check` frontmatter key
- Both functions are exported from `src/card.ts`
- `IssueKind`, `SuggestedAction`, `IntegrityIssue`, `ComplianceIssue`, `IntegrityReport` types added to `src/types.ts`
- `OrchestratorConfig.integrityIntervalSeconds` field added to `src/types.ts`
