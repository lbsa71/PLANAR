# 6.2 Integrity Checker [DONE]

## Description
`src/integrity.ts` — the core integrity engine. Runs two categories of checks
against a set of discovered cards and writes results back to card files.

**Structural checks** (tree consistency):
- Parent-child link symmetry: parent exists, lists this card; child points back
- Status consistency: a DONE node must not have non-DONE children

**Codebase compliance** (verifiable claims):
- Every manifest file entry must exist on disk
- No two cards may claim the same file (duplicate ownership)
- Optionally: source files under `src/` not claimed by any card are flagged

## Acceptance Criteria
- `checkTreeIntegrity(cards, cwd, options)` returns an `IntegrityReport` with `cardIssues` and `complianceIssues`
- `applyIntegrityResults(report, cards, {regressProblematic}, fs)` updates every card file: stamps `last-integrity-check`, prepends revision entry, optionally regresses status
- `formatIntegrityReport(report)` returns a human-readable string suitable for CLI output
- `walkDir` skips `node_modules`, `dist`, `.git`, hidden directories
- Injected `fs.existsSync` is used for manifest file checks (testable)
- `suggestedAction` is `regress-to-plan` for broken child links and status inconsistencies; `regress-to-review` for DONE cards with missing manifest files
