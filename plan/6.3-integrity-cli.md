# 6.3 Integrity CLI & Orchestrator Integration [DONE]

## Description
Wire integrity checking into the CLI and orchestrator.

**CLI** (`planar integrity`):
- Discovers cards, runs `checkTreeIntegrity`, prints `formatIntegrityReport`
- Always stamps `last-integrity-check` and appends revision history on every card
- `--regress` flag: regress DONE cards with issues to PLAN/REVIEW
- `--no-scan-src`: skip unowned file detection
- `--src-dir <dir>`: override source directory (default: `src`)

**Orchestrator** (`planar orchestrate --integrity-interval <secs>`):
- Runs periodic integrity check during orchestration when interval > 0
- Always regresses problematic cards when running automatically
- Logs summary to dashboard
- Adds revision history entry when programmatically changing card status:
  - `propagateConflicts`: adds entry when writing CONFLICTS-WITH status
  - `regressCommonParent`: adds entry when regressing node to PLAN
  - `reapAgent`: detects status change after agent finishes, adds revision entry

## Acceptance Criteria
- `planar integrity [plan-dir]` command works end-to-end
- `planar integrity --regress` regresses flagged cards
- `planar orchestrate --integrity-interval 300` runs checks every 5 minutes
- Orchestrator revision entries are written on conflict propagation, regression, and agent status changes
- `planar --help` shows `integrity` command and all new options
